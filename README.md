# 关于华为开源项目的说明

首先，感谢广大开发者对华为开源项目的关注。针对 GitHub 上出现的部分负面反馈，我们希望在此能够对大家的疑问进行统一答复和说明。

## 开源项目属性说明

华为是一家全球领先的商业科技公司，其核心使命是为客户提供卓越的解决方案和产品服务。需要明确一点，华为作为一家商业公司，并没有必须开源的义务。我们公司所涉及的开源项目，仅仅是部分员工在符合公司政策的前提下，**以个人的技术热情和对开源社区的贡献精神自发推动的行为**。

**开源项目的具体问题**，如代码质量、逻辑错误等，均为员工在开源活动中的个人贡献，可能存在不足之处。这并不能完全代表华为的商业技术能力或整体研发水平。

## 开源初衷

我们支持员工参与开源社区的核心初衷是：
1. 推动技术交流和进步。
2. 为全球开源社区贡献力所能及的资源。
3. 提升技术发展生态的多样性，回馈技术社区。

这并不意味着员工的个人行为能够直接代表公司的标准化研发流程或核心业务能力。华为作为一家负责任的公司，与开源社区一直保持友好合作态度，但愿意支持不等于完全包揽或替所有项目担负责任。

## 关于代码质量反馈

我们十分重视社区对项目的反馈。对于有人指出的问题，无论是代码错误、逻辑问题还是其他不合理之处，如果确实属实，都是改进方向的良好指引。我们希望大家能以建设性的态度多提意见，帮助改善这些项目所能带来的影响。

与此同时，也希望大家理解以下几点：
- 开源项目可能是业余时间的贡献行为，和公司内部产品研发流程是**完全独立**的。
- 我们无法强制要求每位员工的个人开源活动都达到专业级产品的质量水平。
- 刻意攻击或甚至通过不友好语言侮辱他人，并不利于社区环境的良性发展。

## 对社区的期待

开源社区是一个开放的技术交流平台，我们希望开发者之间能够秉持尊重与共赢的态度，围绕技术共同进步。华为支持开源，也希望通过员工的贡献为社区带去价值，但社区成员之间需要有更包容和理解的态度。

如有任何问题，欢迎大家通过 GitHub Issues 提出。我们会根据实际情况给予改进，同时努力让我们的开源项目在能力范围内更符合社区的期待。

## 结语

开源是一种精神，也是一份责任。我们愿意继续在技术上探索，并通过员工参与贡献社区。但市场对整个华为的商业实力和能力评判，需要基于公司的整体表现，而不是基于个人在开源项目上的贡献情况。

期待与大家共同建设一个健康的开源生态系统。
